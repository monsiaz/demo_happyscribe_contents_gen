What are SRT and VTT formats?

SRT, or SubRip Text, is a simple text file that stores subtitles with timecodes. It is widely used for offline media players and basic editing tools. VTT, or WebVTT, is designed for HTML5 video. It adds support for styling cues and is the standard for modern browsers and video platforms.

Key differences:

  • SRT uses hours:minutes:seconds,milliseconds
  • VTT uses hours:minutes:seconds.milliseconds
  • VTT lets you add classes and voice tags

Why you might need to convert SRT to VTT

Moving from SRT to VTT ensures your captions work on websites and streaming services that rely on HTML5. By switching formats you can:

  • enable styling like font, color and position
  • improve compatibility with web players
  • use advanced caption features via Subtitle editor

How to convert SRT to VTT with FFMPEG

FFmpeg is a powerful command-line tool that can convert many media formats. To convert a single file from SRT to VTT use:

ffmpeg -i subtitles.srt subtitles.vtt

For a batch of files on Mac or Linux:

for file in *.srt; do
  ffmpeg -i "$file" "${file%.srt}.vtt"
done

How to convert SRT to VTT using python scripts

A quick Python snippet can read an SRT and write a VTT:

import pysrt

subs = pysrt.open('subtitles.srt')
with open('subtitles.vtt', 'w') as vtt:
  vtt.write('WEBVTT\n\n')
  for idx, sub in enumerate(subs, 1):
    start = sub.start.to_time().isoformat()[:-3]
    end = sub.end.to_time().isoformat()[:-3]
    vtt.write(f'{idx}\n{start} --> {end}\n{sub.text}\n\n')

This script uses the pysrt library to handle timecode parsing. You can tweak it for more style tags or add a caption generator for custom cues.
