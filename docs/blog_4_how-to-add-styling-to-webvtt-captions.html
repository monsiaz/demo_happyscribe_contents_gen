How to add styling to WebVTT captions

Welcome! Today you’ll learn how to add styling to WebVTT captions for a cleaner, more on-brand look. We’ll cover everything from basic conversion tips—like how to convert srt to vtt—to advanced cue styling and integration with HappyScribe tools. By the end, you’ll have all the steps you need to make your subtitles pop on any video player.

What is WebVTT and why style captions?

Web Video Text Tracks (WebVTT) is the standard caption format for HTML5 video. It is a text file that pairs time codes with caption text. By default, most players render captions in their standard style. Adding custom styling helps you:

  • Match your brand colors and fonts
  • Improve readability across devices
  • Highlight key dialogue or speaker names
  • Control positioning for crowded video scenes

Best-practice tip: Keep your style definitions compact. Unnecessary CSS rules can slow down caption rendering on some players.

How do I convert SRT to VTT?

If you already have an SRT file, converting it to VTT is the first step. Here are three simple options:

1. Use FFMPEG for quick conversion

ffmpeg -i subtitles.srt subtitles.vtt
  • Pros: One-line command, fast
  • Pitfall: Older versions of FFMPEG may not handle special characters correctly

2. Try the webvtt-py library

pip install webvtt-py
webvtt convert subtitles.srt --output subtitles.vtt
  • Pros: Works on any system with Python
  • Tip: Use --speaker-break to mark speaker changes

3. Leverage the HappyScribe API

If you prefer an automated flow, HappyScribe’s API can convert your SRT to VTT at scale. Here’s a sample call:

What basic style options does WebVTT support?

WebVTT lets you add a style block at the top of your file. This block uses CSS-like rules for cue classes.

WEBVTT

STYLE
::cue {
  color: white;
  background-color: rgba(0, 0, 0, 0.7);
  font-size: 1.2em;
}

::cue(.speaker1) {
  color: #FF5722;
}

::cue(.speaker2) {
  color: #03A9F4;
}
  • color: Text color in standard CSS hex or keyword
  • background-color: Use rgba for transparency
  • font-size: Relative units like em or rem adapt across screens

Pro tip: Group similar cues by adding custom class names in the cue settings section. This keeps your style rules DRY and easy to update.

How can I control cue position and alignment?

Beyond style, you can fine-tune where each caption appears:

00:00:05.000 --> 00:00:10.000 align:start line:90% position:10%
This text appears near the bottom left.

00:00:15.000 --> 00:00:20.000 align:center line:80% position:50%
Centered at the bottom.
  • align: start, center, end
  • line: Percent or number of lines from top or bottom
  • position: Horizontal placement from left edge

Pitfall: Some players ignore advanced cues. Always test on Chrome, Firefox, and Safari. For wide coverage, rely on core CSS styling in the STYLE block.

What are common best practices when styling WebVTT?

  • Keep contrast high: White text on dark background or vice versa
  • Use relative units for font size: 1.2em scales with user settings
  • Limit the number of custom classes: Too many selectors can cause slow parsing
  • Test on mobile: Small screens need larger text and more padding

How do I integrate styled VTT with HappyScribe API?

After you’ve converted your file, you might want to host or serve it via API calls. Here is a basic integration flow:

// 1. Convert SRT to VTT
POST https://api.happyscribe.com/v1/convert

// 2. Add styling block locally or in storage
// 3. Upload styled VTT back to HappyScribe for hosting

POST https://api.happyscribe.com/v1/subtitles
Headers: Authorization Bearer YOUR_API_KEY
Body: { "file": styled_subtitles.vtt, "video_id": "12345" }
  • Tip: Store your VTT files in S3 and link them via API calls
  • Tip: Use webhooks to get notifications when subtitles are processed
