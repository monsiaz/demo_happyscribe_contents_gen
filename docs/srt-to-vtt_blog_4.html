How to convert SRT to VTT and translate captions

Welcome! In this guide you’ll learn how to convert SRT to VTT, translate captions, and polish them for any video project. Whether you want a simple command-line solution or a cloud-based approach with the HappyScribe API, you’ll find step-by-step examples, best-practice tips, and pitfalls to avoid. Let’s dive in!

How do I convert SRT to VTT with FFMPEG?

FFMPEG is a free, open source tool that can handle almost any media task, including converting subtitle formats. Follow these steps to change SRT to VTT in seconds.

1. install FFMPEG

First, make sure FFMPEG is on your system. On macOS you can use Homebrew:

brew install ffmpeg

On Ubuntu or Debian:

sudo apt update  
sudo apt install ffmpeg

2. run the conversion command

Once installed, run this simple command:

ffmpeg -i input.srt output.vtt

This reads input.srt and writes a valid WebVTT file output.vtt. You can batch-process multiple files with a quick shell loop:

for f in *.srt; do  
  ffmpeg -i "$f" "${f%.srt}.vtt"  
done

Tip: check encoding

If your SRT uses a non-UTF8 encoding you may see odd characters. Convert it first:

iconv -f ISO-8859-1 -t UTF-8 input.srt -o input-utf8.srt

Pitfall to avoid

  • Don’t forget the .vtt extension. Missing it can confuse players.
  • Make sure each subtitle line ends with a blank line to keep timing intact.

What’s the fastest way to convert SRT to VTT with webvtt-py?

If you prefer Python, webvtt-py is a handy package for reading and writing VTT files. You can also modify text on the fly.

1. install the package

pip install webvtt-py

2. convert with a script

Create a file named convert.py:

import webvtt

for caption in webvtt.read('input.srt'):
    # optional: caption.text = caption.text.upper()
    caption.save('output.vtt')

Run it:

python convert.py

Best-practice tip

  • Use caption.text to clean up line breaks or apply simple formatting.
  • If you need batch support, wrap webvtt.read in a loop.

Pitfall to avoid

By default webvtt-py expects a VTT. Make sure you load your SRT correctly. If you see parsing errors, open the file in a text editor and remove odd characters.

How can I translate captions using the HappyScribe API?

Translating captions by hand can be slow. Our HappyScribe API can auto-translate any SRT or VTT file to your target language. Here’s a quick example in Python.

1. get your API key

Sign in to your HappyScribe account and find the API section. Keep your key safe.

2. install the HTTP client

pip install requests

3. translate captions

import requests

api_url = "https://api.happyscribe.com/v1/jobs"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "content": open("input.srt").read(),
    "source_language": "en",
    "target_language": "es",
    "format": "srt"
}

response = requests.post(api_url, json=data, headers=headers)
result = response.json()

# Wait for the job to finish, then download
job_id = result["id"]
download_url = f"{api_url}/{job_id}/download"
translated = requests.get(download_url, headers=headers).text
with open("output.es.srt", "w", encoding="utf-8") as f:
    f.write(translated)

This sample sends your English SRT to HappyScribe and returns a Spanish SRT. You can change format to vtt to get a VTT file straight away.

Best-practice tip

  • Check the translated text with our Subtitle editor to catch any unnatural phrasing.
  • Use Human-made subtitles if you need broadcast-quality translations.

Pitfall to avoid

Don’t forget to respect rate limits. Batch your requests in groups of 10 to avoid hitting the cap.
