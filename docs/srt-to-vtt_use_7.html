Event streaming provider: convert SRT to VTT with HappyScribe

Are you an event streaming provider looking to deliver high-quality captions across every platform and player? In this guide, you’ll learn how to convert SRT files to VTT format in a few simple steps. You’ll see how HappyScribe’s API and tools can streamline your workflow, saving you time and making your live events more accessible to audiences around the world.

why SRT to VTT matters for your streaming platform?

Most web video players and HTML5 frameworks natively support WebVTT (VTT). If you start with an SRT file, your captions might not display correctly in modern players or across different devices. By converting to VTT, you ensure:

  • Full compatibility with HTML5 video and popular players like Video.js and JW Player
  • Support for styling tags, colors, and WebVTT cues
  • Improved accessibility for viewers who rely on captions
  • Consistent timing and position information

Imagine you’ve just hosted a live keynote and you need captions on your replay page. The speaker’s script is in SRT. With a quick conversion to VTT, you’ll be ready to deliver a polished, accessible video right away.

how can you set up a reliable SRT to VTT workflow?

We’ll walk you through a step-by-step workflow that you can integrate into your existing event streaming pipeline. Whether you prefer command-line tools, server scripts, or direct API calls, you’ll find an option that works for your setup.

step 1: gather your SRT files

Start by collecting the SRT transcripts from your captioning provider or your in-house team. Make sure your files have consistent naming and follow a simple convention like:

  • session1.srt
  • session2.srt
  • session3.srt

step 2: choose your conversion method

Here are three common ways to convert SRT to VTT:

  • Command-line tool: Use an open source script or npm package to convert locally
  • Custom server script: Write a small Node.js or Python script that calls an API
  • HappyScribe API: Let HappyScribe take care of conversion on our servers and return a ready-to-use VTT file

step 3: command-line conversion (optional)

If you’d rather run a quick conversion on your own machine, you can use an npm package like srt-to-vtt. First, install it:

npm install -g srt-to-vtt

Then convert your file:

srt-to-vtt session1.srt session1.vtt

This works well for one-off tasks, but when you have dozens of sessions, it’s time to automate.

step 4: automate with a Node.js script

Create a simple script that reads SRT files from a folder, sends them to HappyScribe for conversion, and downloads the VTT. You’ll need an API key from your HappyScribe dashboard.

const axios = require('axios');
const fs = require('fs');
const path = require('path');

const API_KEY = 'YOUR_HAPPYSCRIBE_API_KEY';
const folderPath = './subtitles';

async function convertFile(fileName) {
  const filePath = path.join(folderPath, fileName);
  const fileStream = fs.createReadStream(filePath);

  // 1. Upload the SRT file
  const uploadRes = await axios.post(
    'https://api.happyscribe.com/v2/transcriptions',
    fileStream,
    {
      headers: {
        'Authorization': `Bearer ${API_KEY}`,
        'Content-Type': 'text/plain'
      },
      params: { format: 'vtt' }
    }
  );

  const transcriptId = uploadRes.data.id;

  // 2. Poll until the conversion is ready
  let status = 'processing';
  while (status !== 'done') {
    await new Promise(r => setTimeout(r, 2000));
    const statusRes = await axios.get(
      `https://api.happyscribe.com/v2/transcriptions/${transcriptId}`,
      { headers: { 'Authorization': `Bearer ${API_KEY}` } }
    );
    status = statusRes.data.status;
  }

  // 3. Download the VTT file
  const downloadRes = await axios.get(
    `https://api.happyscribe.com/v2/transcriptions/${transcriptId}/subtitles.vtt`,
    {
      headers: { 'Authorization': `Bearer ${API_KEY}` },
      responseType: 'stream'
    }
  );
  const outPath = path.join(folderPath, fileName.replace('.srt', '.vtt'));
  downloadRes.data.pipe(fs.createWriteStream(outPath));
  console.log(`Converted ${fileName} to VTT`);
}

fs.readdirSync(folderPath)
  .filter(f => f.endsWith('.srt'))
  .forEach(f => convertFile(f));

Save this as convert.js, run node convert.js, and watch your folder fill up with VTT files.

step 5: integrate into your CI/CD or streaming pipeline

Drop this script in your build process, your Docker image, or your serverless function. Each time you upload new SRT files, the script runs automatically and leaves you with VTT files ready for your player.
