Understanding SRT and VTT subtitle formats

Key differences between SRT and VTT

SRT files consist of numbered cues with start and end timecodes, followed by plain text. VTT files share a similar structure but add support for styling, positioning, and additional metadata. For example:

00:00:05,000 --> 00:00:07,000
Hello world.

becomes

WEBVTT

00:00:05.000 --> 00:00:07.000 align:start position:0%
Hello world.

Notice the header WEBVTT, millisecond separators with periods, and optional cue settings like align and position.

When to use VTT over SRT

  • HTML5 video embedding: Most browsers expect .vtt files for the <track> tag.
  • Styling control: VTT cues support CSS-like classes for fonts, colors, and backgrounds.
  • Accessibility compliance: Screen readers and assistive tech often prefer VTT metadata.

Tip: Always validate your converted VTT with a linter or the Subtitle editor to ensure cue syntax is correct before deployment.

Using FFMPEG to convert srt to vtt

Basic FFMPEG command example

FFMPEG provides a quick binary-based conversion without external libraries:

ffmpeg -i subtitles.srt subtitles.vtt

This command reads subtitles.srt and outputs a valid subtitles.vtt. FFMPEG auto-converts timecode delimiters and adds the WEBVTT header.

Advanced FFMPEG options

If you need to adjust cue style or encoding, consider:

ffmpeg -i subs.srt -c:s text -metadata:s:s:0 language=eng subtitles.vtt
  • -c:s text ensures subtitle codec is plain text.
  • -metadata:s:s:0 language=eng tags the VTT with the correct language metadata.

Best practices and pitfalls

  • Whitespace sensitivity: Extra blank lines in SRT can break conversion. Always trim trailing spaces.
  • Encoding matters: Ensure your SRT is UTF-8 to prevent garbled characters in VTT.
  • File naming: Match the naming convention video.vtt with your video file video.mp4 for seamless HTML5 integration.

Converting multiple files with webvtt-py

Installing and setting up webvtt-py

webvtt-py is a Python library designed specifically for VTT manipulation. To install:

pip install webvtt-py

Then import it in your script:

import webvtt

Batch conversion script

Here is a Python script to convert all SRT files in a folder:

import os
import webvtt

SOURCE_DIR = 'srt_files'
OUTPUT_DIR = 'vtt_files'
os.makedirs(OUTPUT_DIR, exist_ok=True)

for filename in os.listdir(SOURCE_DIR):
    if filename.lower().endswith('.srt'):
        srt_path = os.path.join(SOURCE_DIR, filename)
        vtt_path = os.path.join(OUTPUT_DIR, filename.rsplit('.', 1)[0] + '.vtt')
        webvtt.from_srt(srt_path).save(vtt_path)
        print(f'Converted {srt_path} to {vtt_path}')

This loop automatically reads each .srt, converts it, and writes a .vtt file in the target folder.

Tip: handling special characters

Pitfall: SRT files may contain HTML entities or special UTF-8 characters. Before running the batch job, strip or replace unsupported tags:

with open(srt_path, 'r', encoding='utf-8') as f:
    text = f.read().replace('&', '&')
with open(srt_path, 'w', encoding='utf-8') as f:
    f.write(text)

This ensures clean conversion to VTT without unexpected artifacts.

Automating conversion with HappyScribe API

Getting started with the HappyScribe API

The HappyScribe API allows you to upload media, request Automatic transcription or Human-made subtitles, and retrieve results in multiple formats including SRT and VTT. First, obtain your API key in the API documentation.

export HAPPYSCRIBE_API_KEY="your_api_key_here"

Example Python script

Below is a sample script that automates upload, conversion, and download of VTT subtitles:

import requests
import time

API_URL = 'https://api.happyscribe.com/v1'
API_KEY = 'your_api_key_here'
FILE_PATH = 'video.mp4'

# Step 1: Upload media
files = {'file': open(FILE_PATH, 'rb')}
headers = {'Authorization': f'Bearer {API_KEY}'}
upload_resp = requests.post(f'{API_URL}/transcriptions', headers=headers, files=files)
job_id = upload_resp.json()['id']
print(f'Transcription requested: {job_id}')

# Step 2: Poll for completion
status = ''
while status != 'completed':
    time.sleep(10)
    status_resp = requests.get(f'{API_URL}/transcriptions/{job_id}', headers=headers)
    status = status_resp.json()['status']
    print(f'Status: {status}')

# Step 3: Download VTT subtitles
download_resp = requests.get(f'{API_URL}/transcriptions/{job_id}/subtitles?format=vtt', headers=headers)
with open('output.vtt', 'wb') as f:
    f.write(download_resp.content)
print('Downloaded output.vtt')

This script leverages HappyScribe’s API to streamline bulk workflows, handing off accuracy to Automatic transcription or the option for Human-made subtitles.

Pitfalls and error handling

  • Timeouts: Large media uploads may require chunked uploads or increased timeouts.
  • Rate limits: Respect API rate limits and implement exponential backoff on HTTP 429 responses.
  • Validation: Always inspect the returned VTT in a Subtitle editor for minor timing tweaks.

Enabling styling and accessibility in VTT

Using CSS and Regions

VTT supports CSS classes within cues. For example:

00:00:10.000 --> 00:00:12.000 class:highlight
Hello world

Define the styling in your HTML:

<style>
.vtt-cue.highlight { color: yellow; background: rgba(0,0,0,0.7); }
</style>

Defining regions for complex layouts

Regions allow you to place subtitles in specific screen areas:

WEBVTT
Region: top-right id=TR width=40% lines=3 regionanchor=0%,0% viewportanchor=100%,0%

00:00:05.000 --> 00:00:07.000 region:TR
Multi-line note in top right.

Best practice: Test on multiple devices to ensure readability and avoid overlap with UI elements.
