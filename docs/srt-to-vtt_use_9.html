Accessibility specialist use case: convert SRT to VTT

As an accessibility specialist, you know that making video content available to all users is a core part of your role. You work with captions every day to meet legal standards, improve user satisfaction, and open content to viewers with hearing impairments or non-native speakers. One key step in your daily routine is converting subtitle files from SRT to VTT format. With HappyScribe, you can speed up that task, maintain accuracy, and keep your projects on schedule.

Why does this conversion matter for your work?

When you deliver caption files to developers, content producers, or learning platforms, they often require WebVTT (VTT) format. It’s the standard for HTML5 video players. If you only have SRT files, you need to convert them quickly and reliably—especially when you’re handling dozens of videos each week.

  • Stay compliant with accessibility guidelines like WCAG 2.1
  • Ensure captions render properly in modern web players
  • Avoid last-minute fixes before a launch or a live event
  • Save time by automating repetitive tasks

What tools do you need to convert SRT to VTT?

To set up a smooth conversion process, you’ll need:

  • A HappyScribe account with API access
  • Your SRT files stored on a local drive or cloud storage
  • A simple script in your preferred language (we’ll show a Node.js example)
  • Basic knowledge of API calls via cURL or code

What is the step-by-step workflow?

  1. Create your HappyScribe API key

    Log in to your HappyScribe dashboard, go to the API section, and copy your secret key. You’ll use this to authenticate each call.

  2. Install dependencies

    For Node.js, you need the axios package to make HTTP requests:

    npm install axios
  3. Write a conversion script

    Create a file named convert.js and add the following code:

    const axios = require('axios');
const fs = require('fs');
const path = require('path');

async function convertSrtToVtt(inputPath, outputPath) {
  const apiKey = 'YOUR_HAPPYSCRIBE_API_KEY';
  const fileData = fs.readFileSync(inputPath, 'utf-8');

  // Send the SRT file to HappyScribe for conversion
  const response = await axios.post(
    'https://api.happyscribe.co/v1/conversions',
    {
      type: 'srt_to_vtt',
      content: fileData
    },
    {
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      }
    }
  );

  // Save the VTT output
  fs.writeFileSync(outputPath, response.data.content);
  console.log('Conversion complete:', outputPath);
}

const inputFile = path.join(__dirname, 'captions.srt');
const outputFile = path.join(__dirname, 'captions.vtt');

convertSrtToVtt(inputFile, outputFile).catch(console.error);
  4. Run your script

    In your terminal, execute:

    node convert.js

    After a moment, you’ll see a success message and find the VTT file in your folder.

  5. Integrate with your video player

    Link the new .vtt file in your HTML5 player:

    <video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="captions.vtt" srclang="en" label="English">
</video>

    Your captions will now appear correctly in Chrome, Firefox, Safari, and Edge.
