Podcast producer use case: SRT to VTT

In this use case we’ll show you how a podcast producer can convert SRT files to VTT in minutes with HappyScribe. If you’re creating video clips for social media, embedding transcripts on your site or powering a web player with captions, you need the right format. We’ll guide you through the why and the how, share a step-by-step workflow, point out key features and wrap up with answers to your top questions.

What’s the scenario?

Imagine you’re wrapping up a new episode and you’ve got an SRT file from your caption service. Now you want to:

  • Publish captions on your website player that only accepts WebVTT (VTT).
  • Meet accessibility standards for hearing-impaired listeners.
  • Boost SEO by embedding searchable transcripts in HTML.
  • Create dynamic video quotes with time-coded subtitles for social channels.

While SRT is great, most modern players and web apps prefer VTT. Manually converting your file can take time and risk errors in timestamps. That’s where HappyScribe steps in. You’ll learn how to go from SRT to VTT with a few clicks or a single API call, saving you hours of manual work.

How does the conversion workflow look?

We’ve broken down the process into clear steps. You can follow along in our web app, or plug into HappyScribe’s API if you want to build an automated pipeline. We’ll cover both paths.

  3. Upload your SRT file.

Automated workflow with the HappyScribe API

For power users and developers, here’s how to build a simple script that:

  • Uploads your SRT file.
  • Triggers a conversion job.
  • Polls for completion.
  • Downloads your VTT file automatically.

We’ll give you a sample in Node.js. Feel free to adapt to Python, Ruby or any language that can make HTTP calls.

// install axios: npm install axios
const axios = require('axios');
const fs = require('fs');
const apiKey = 'YOUR_HAPPYSCRIBE_API_KEY';

async function convertSrtToVtt(inputPath, outputPath) {
  // 1. Upload SRT
  const formData = new FormData();
  formData.append('file', fs.createReadStream(inputPath));
  formData.append('source_type', 'subtitle');
  formData.append('format', 'srt');

  const uploadRes = await axios.post('https://api.happyscribe.com/v1/uploads', formData, {
    headers: formData.getHeaders({ Authorization: `Bearer ${apiKey}` })
  });
  const fileId = uploadRes.data.id;

  // 2. Create conversion job
  const jobRes = await axios.post(
    'https://api.happyscribe.com/v1/conversions',
    {
      upload_id: fileId,
      source_format: 'srt',
      target_format: 'vtt'
    },
    { headers: { Authorization: `Bearer ${apiKey}` } }
  );
  const jobId = jobRes.data.id;

  // 3. Poll for job status
  let status = 'processing';
  while (status === 'processing') {
    await new Promise(r => setTimeout(r, 2000)); // wait 2 seconds
    const statusRes = await axios.get(`https://api.happyscribe.com/v1/conversions/${jobId}`, {
      headers: { Authorization: `Bearer ${apiKey}` }
    });
    status = statusRes.data.status;
  }

  // 4. Download VTT
  const downloadUrl = statusRes.data.download_url;
  const fileRes = await axios.get(downloadUrl, { responseType: 'stream' });
  const writer = fs.createWriteStream(outputPath);
  fileRes.data.pipe(writer);
  return new Promise((resolve, reject) => {
    writer.on('finish', resolve);
    writer.on('error', reject);
  });
}

convertSrtToVtt('episode123.srt', 'episode123.vtt')
  .then(() => console.log('Conversion complete!'))
  .catch(err => console.error(err));

With this script, you can drop your SRT file into a watched folder and get back a VTT without lifting a finger. That’s perfect for batch jobs or integrating into your CI/CD pipeline for podcast production.
