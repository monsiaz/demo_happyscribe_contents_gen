How to convert SRT to VTT using the HappyScribe API

In this section, we will send your SRT file to the API and pull back a VTT file. You can use curl, Node.js, or Python. Let’s explore each.

Using curl for quick tests

curl -X POST https://api.happyscribe.com/v1/subtitle-files \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@/path/to/subtitles.srt" \
  -F "from_format=srt" \
  -F "to_format=vtt" \
  -o subtitles.vtt

This single command does it all: uploads your SRT, converts it, and saves a VTT locally.

Node.js example for batch processing

const fs = require('fs');
const axios = require('axios');

async function convertSrtToVtt(inputPath, outputPath) {
  const fileStream = fs.createReadStream(inputPath);
  const form = new FormData();
  form.append('file', fileStream);
  form.append('from_format', 'srt');
  form.append('to_format', 'vtt');

  const response = await axios.post(
    'https://api.happyscribe.com/v1/subtitle-files',
    form,
    {
      headers: {
        ...form.getHeaders(),
        Authorization: 'Bearer YOUR_API_KEY'
      },
      responseType: 'stream'
    }
  );

  response.data.pipe(fs.createWriteStream(outputPath));
  console.log('Converted:', outputPath);
}

convertSrtToVtt('example.srt', 'example.vtt');

Best-practice tip: Wrap your API calls in try/catch. If you have dozens of files, loop over them and await each conversion to avoid hitting rate limits.

Python example with requests

import requests

def convert_srt_to_vtt(input_file, output_file, api_key):
    url = 'https://api.happyscribe.com/v1/subtitle-files'
    with open(input_file, 'rb') as f:
        files = {
            'file': f,
            'from_format': (None, 'srt'),
            'to_format': (None, 'vtt')
        }
        headers = {'Authorization': f'Bearer {api_key}'}
        response = requests.post(url, files=files, headers=headers, stream=True)
        response.raise_for_status()
        with open(output_file, 'wb') as out:
            for chunk in response.iter_content(chunk_size=8192):
                out.write(chunk)
    print('Done converting', input_file)

convert_srt_to_vtt('example.srt', 'example.vtt', 'YOUR_API_KEY')

How to verify and edit your VTT files

After conversion, you want to ensure timing and text line breaks look right. Here are two ways:

  • Use webvtt-py to quickly scan through cues:
import webvtt

for caption in webvtt.read('example.vtt'):
    print(caption.start, caption.text)

If you spot overlaps or missing cues, adjust them directly in the VTT or redo with custom parameters through your Subtitle editor.

  • Load your VTT in a browser or video player. Play a snippet and verify each caption aligns with speech.
  • Open the file in a GUI editor for quick text edits or bulk find-and-replace.

Pitfall to avoid: Some players need the WEBVTT header on top of the file. Confirm it starts with WEBVTT and a blank line. The API handles this automatically, but custom scripts may drop it.

Common pitfalls and best practices

When you work with subtitle conversions, watch out for these issues.

  • Time format mismatches: Ensure your source SRT uses the format HH:MM:SS,ms. If not, FFMPEG can normalize it.
  • Rate limits: If you batch dozens of files, space out requests or handle 429 errors by pausing and retrying.
  • Encoding errors: Always save your SRT files in UTF-8. Nonstandard characters can break the conversion.
  • Line length: VTT players often wrap long lines. Keep captions under 80 characters for readability.
