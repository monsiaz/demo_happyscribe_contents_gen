How to convert SRT to VTT manually using a text editor

In this article you’ll learn how to convert srt to vtt using only a text editor and a handful of simple steps. We’ll dive into why you might need this format swap, walk through manual edits, share tips to avoid common mistakes, and give you quick ways to automate with tools like FFMPEG, webvtt-py, or the HappyScribe API. Ready? Let’s get started!

What is SRT and VTT and why you might need conversion?

SRT and VTT are both subtitle file formats. SRT is older and widely supported. VTT is newer, built for web environments, and required by HTML5 video players. If you have a subtitle file from an editor or transcription service like HappyScribe Subtitle Editor, converting srt to vtt makes it ready for online players.

  • SRT: Simple time stamps using commas (00:00:01,500).
  • VTT: Requires a header and uses dots in time stamps (00:00:01.500).
  • WebVTT supports styling, positioning, and metadata.

Best practice tip: Always back up your original SRT file before you edit. That way you can restart if something goes wrong.

How to prepare your SRT file for conversion

Before making any changes, open your .srt file in a text editor (Notepad, VSCode, Sublime Text). Look out for these points:

Are all time codes properly formatted?

  • Each line should follow HH:MM:SS,mmm --> HH:MM:SS,mmm.
  • Your editor’s line endings can affect formatting—stick to LF or CRLF consistently.

Do you have extra blank lines or notes?

  • Remove any lines that start with “NOTE” or editor comments.
  • Ensure sequence numbers are in order: 1, 2, 3…

Pitfall to avoid: Extra spaces before or after the arrow (-->) can break a VTT parser. Keep it tight and consistent.

Manual steps to convert SRT to VTT

This is the bread and butter method. It takes less than five minutes.

  1. Open the SRT file in your text editor.
  2. Add the VTT header at the very top:
WEBVTT
  1. Convert all commas in timecodes to dots. In VSCode or Sublime, use find and replace:
Find:    (\d{2}:\d{2}:\d{2}),(\d{3})
Replace: $1.$2
Use regex mode
  1. Save the file with a .vtt extension.

That’s it. You now have a valid VTT file ready for HTML5 video. Test with:

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track label="English" kind="subtitles" srclang="en" src="subtitles.vtt" default>
</video>

Best practice tip: Always test your VTT in multiple browsers. Some may skip cues if there’s a tiny format issue.

How to use FFMPEG for fast conversion?

If you have FFMPEG installed, converting srt to vtt is a one-line command:

ffmpeg -i input.srt output.vtt

FFMPEG will auto-detect the input subtitles, add the WEBVTT header, and swap commas for dots. It works well for most files.

  • If you see encoding issues, add -sub_charenc UTF-8.
  • To burn subtitles directly into a video, use -vf subtitles=input.srt.

Pitfall to avoid: FFMPEG might not keep styling tags. If you need CSS positioning, use a manual edit or webvtt-py.

How to use webvtt-py for quick scripting?

webvtt-py is a Python library that reads SRT and outputs VTT. Here’s a sample script:

from webvtt import WebVTT

# Load and save
vtt = WebVTT().read('input.srt')
vtt.save('output.vtt')

Install it with:

pip install webvtt-py
  • Use this in batch scripts to convert multiple files.
  • Works on Windows, Mac, and Linux.

Best practice tip: Check the library version to avoid bugs. If you run into errors, upgrade with pip install --upgrade webvtt-py.
