E-learning developer: convert SRT to VTT for seamless captioning

As an e-learning developer, you know how critical accurate, accessible captions are for your online courses. whether you’re building a compliance training for a global workforce or launching a language learning series, the right subtitle format can make all the difference. in this guide, you’ll learn why converting SRT to VTT matters for modern platforms, how to set up a smooth workflow with HappyScribe, and which key features will save you time and keep your learners engaged.

why does SRT to VTT conversion matter for e-learning?

you might already have SRT files from past projects, client submissions, or auto-generation tools. but most modern video players and learning management systems (LMS) prefer WebVTT (.vtt) because it supports:

  • styled captions (colors, positioning)
  • chapter markers for long lectures
  • better browser compatibility
  • HTML5 video integration

if you don’t convert, your captions might not appear, or you’ll lose styling and timing accuracy. every playback glitch can distract learners and hurt completion rates.

how can you set up an efficient conversion workflow?

we’ll walk through a simple, step-by-step workflow using HappyScribe tools and API calls. by the end, you’ll have a repeatable process to convert large batches of SRT files into VTT, customize captions, and push them to your LMS.

1. collect your SRT files

start by organizing all SRT files into a single folder. you can:

  • download exports from transcription services
  • gather client-provided captions
  • auto-generate SRT using HappyScribe’s subtitle editor or other tools

2. set up your environment

you’ll need:

  • a command-line tool that supports cURL or HTTP requests
  • your HappyScribe API key (you can get one on your dashboard under API settings)
  • the folder path for your SRT files

tip: store your API key as an environment variable (e.g., HAPPYSCRIBE_API_KEY) to keep your scripts secure.

3. write the conversion script

below is a sample Bash script. it loops through all SRT files in a folder, sends them to the HappyScribe API, and finally downloads the converted VTT files.

# set the source and target folders
SRC_DIR="./srt_files"
DEST_DIR="./vtt_files"

# create destination folder if needed
mkdir -p "$DEST_DIR"

# loop through each .srt file
for file in "$SRC_DIR"/*.srt; do
  filename=$(basename -- "$file" .srt)
  echo "processing $filename.srt …"

  # upload and convert
  response=$(curl -s -X POST "https://api.happyscribe.com/v1/convert" \
    -H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
    -F "file=@$file" \
    -F "source_format=SRT" \
    -F "target_format=VTT")

  # extract conversion ID
  conversion_id=$(echo $response | jq -r '.id')

  # wait for conversion to finish
  status="pending"
  while [ "$status" != "done" ]; do
    sleep 2
    check=$(curl -s -X GET "https://api.happyscribe.com/v1/convert/$conversion_id" \
      -H "Authorization: Bearer $HAPPYSCRIBE_API_KEY")
    status=$(echo $check | jq -r '.status')
    echo "status: $status"
  done

  # download the VTT
  download_url=$(echo $check | jq -r '.result_url')
  curl -s "$download_url" -o "$DEST_DIR/$filename.vtt"
  echo "saved $filename.vtt"
done

echo "all files converted!"

this script:

  • uploads each SRT to the HappyScribe convert endpoint
  • polls the API until conversion is complete
  • downloads the final VTT to your chosen folder
