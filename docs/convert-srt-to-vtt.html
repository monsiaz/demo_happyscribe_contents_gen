Convert SRT to VTT: quick, reliable subtitle conversion

Subtitle file formats vary based on platform requirements, we discuss how to convert SRT to VTT and apply them across web and streaming players.

What are SRT and VTT formats?

SRT, short for SubRip Subtitle, uses plain text cues numbered sequentially with timestamps in HH:MM:SS,mmm format. It is supported widely by media players, video editors, and broadcasting tools. VTT, or Web Video Text Tracks, builds on SRT structure but adds a mandatory WEBVTT header, timecodes in HH:MM:SS.mmm format, and supports positioning, styling, and voice metadata. It is native to HTML5 elements and modern streaming frameworks.

Why convert srt to vtt for modern video workflows?

Most HTML5 and JavaScript players expect WebVTT, so converting ensures compatibility with web delivery and caption generator tools. Whether you use an API or a subtitle editor, switching from SRT to VTT simplifies integration with streaming services and accessibility frameworks, especially when combining Automatic transcription or Human-made subtitles from HappyScribe.

Use HappyScribe Caption generator or API when you need to convert srt to vtt quickly and accurately.

Command-line tools for converting SRT to VTT

Using FFmpeg

FFmpeg can perform a direct conversion:

ffmpeg -i input.srt output.vtt

This adds the WEBVTT header automatically and reformats timestamps. FFmpeg is ideal for batch processing hundreds of files in server environments.

Using Python for manual conversion

For custom workflows you can write a short script:

from pathlib import Path

def srt_to_vtt(path):
    lines = Path(path).read_text().splitlines()
    lines.insert(0, "WEBVTT\n")
    content = []
    for line in lines:
        content.append(line.replace(",", ".") if "-->" in line else line)
    Path(path.with_suffix(".vtt")).write_text("\n".join(content))

srt_to_vtt(Path("subtitle.srt"))

This snippet converts time separators and adds the VTT header. It is useful if you need to handle custom metadata or voice tags before publishing.

How to batch convert SRT to VTT files

  • On Linux or macOS use a shell loop: for f in *.srt; do ffmpeg -i "$f" "${f%.srt}.vtt"; done
  • On Windows PowerShell: Get-ChildItem *.srt | ForEach-Object { ffmpeg -i $_.Name "$($_.BaseName).vtt" }
  • Leverage HappyScribe API to process multiple files in parallel: call the API with track_format set to vtt for batch jobs

Integrating HappyScribe API for automated conversion

Call HappyScribe API to upload an SRT file and retrieve VTT output. Example with curl:

curl -X POST https://api.happyscribe.com/v1/subtitles \

-H "Authorization: Bearer YOUR_TOKEN" \

-F file=@input.srt -F output_format=vtt

The response returns a URL to download the .vtt file. Use this in your CI/CD pipeline or video management system for seamless subtitle workflows.

Manual conversion with text editors and Adobe Premiere

You can open an SRT file in Notepad or any text editor, replace commas with periods in timecodes, and add WEBVTT at the top. For simple tasks you can perform srt to vtt changes in Notepad without external apps. In Adobe Premiere Pro import the SRT as captions then export as WebVTT. This approach works without external tools but is best for one-off tasks or quick edits using the Subtitle editor.

Tips for quality assurance after conversion

  • Use a validator like WebVTT checker to catch mismatches or missing headers
  • Review punctuation and line breaks in Subtitle editor to ensure readability
  • Test captions using a browser element or a media player like VLC
