How do you convert srt to vtt with FFMPEG?

FFMPEG is a popular open source tool for audio and video processing. It can also handle subtitle format changes in a snap.

Install FFMPEG

On macOS with Homebrew:

brew install ffmpeg

On Ubuntu or Debian:

sudo apt update
sudo apt install ffmpeg

Run a simple conversion command

To convert a single file:

ffmpeg -i input.srt output.vtt

Here, -i specifies the input file. FFMPEG reads the SRT and writes a WebVTT file automatically.

Tip: If your SRT uses comma decimals (00:00:01,500), FFMPEG will handle it. Check the output timecodes to be sure they match your source.

How can webvtt-py make batch conversion easier?

The webvtt-py library is a Python package for reading and writing WebVTT files. You can use it to script batch conversions in just a few lines.

Setup your Python environment

python3 -m venv venv
source venv/bin/activate
pip install webvtt-py

Sample batch conversion script

import os
import webvtt

input_folder = 'subs'
output_folder = 'vtt_subs'
os.makedirs(output_folder, exist_ok=True)

for filename in os.listdir(input_folder):
    if filename.lower().endswith('.srt'):
        in_path = os.path.join(input_folder, filename)
        out_name = filename.rsplit('.', 1)[0] + '.vtt'
        out_path = os.path.join(output_folder, out_name)
        webvtt.from_srt(in_path).save(out_path)
        print(f'Converted {filename} to {out_name}')

This script loops through all SRT files in subs and writes VTT files to vtt_subs. Adjust folder names to match your project.

Tip: Make sure your file names do not contain special characters. That can cause errors when creating new files.

How do you call the HappyScribe API for format conversion?

If you already have transcripts in your HappyScribe account, you can ask the API to convert formats without downloading and re-uploading files. This is great when working at scale.

Get your API key

1. Log into your HappyScribe account. API
2. Navigate to your developer settings.
3. Copy your secret key.

Submit a conversion request

import requests

api_key = 'YOUR_API_KEY'
job_id = '12345'
url = f'https://api.happyscribe.com/v1/transcriptions/{job_id}/subtitles'
params = {
    'format': 'vtt'
}
headers = {
    'Authorization': f'Bearer {api_key}'
}

response = requests.get(url, headers=headers, params=params)
if response.status_code == 200:
    with open('subtitle.vtt', 'wb') as f:
        f.write(response.content)
    print('Downloaded VTT file')
else:
    print('Error:', response.status_code, response.text)

This code fetches the VTT subtitle for a given job. You can loop over a list of job IDs to process many files in one go.

Tip: For large batches, add a short delay or handle HTTP errors to avoid hitting rate limits on the HappyScribe API.

What are best practices and pitfalls to avoid?

  • Always check timecode formats. SRT uses commas, VTT uses dots.
  • Validate your VTT files in a player like VLC or in-browser. A broken header can block the whole file.
  • Keep your file names consistent. Avoid spaces and special symbols.
  • Use your Subtitle editor in HappyScribe for quick fixes after conversion.
  • For high accuracy, trust Human-made subtitles when you need perfect timing.

With these pointers, you can convert without surprises and keep your subtitles in sync.
