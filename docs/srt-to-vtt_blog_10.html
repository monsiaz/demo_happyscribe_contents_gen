How to convert SRT to VTT using a Docker container

Welcome! In this guide, you will learn how to convert srt to vtt using a Docker container. Whether you are a content creator, marketer, educator, or accessibility lead, we will walk you through a clear path. You will see real-world tips, common pitfalls, and best practices. Let’s dive in.

Why would you want to convert SRT to VTT?

Before we jump in, let’s look at a few reasons to convert srt to vtt:

  • Browser compatibility: Most modern web video players prefer VTT.
  • Styling support: VTT lets you add cues for text styling and positioning.
  • Accessibility: VTT is the standard for HTML5 captions and improves reader experience.
  • Integration: Popular tools like the Subtitle editor in HappyScribe use VTT.

What is SRT and why pick VTT?

SRT is a simple subtitle format. It uses numbered cues and timestamps. VTT builds on that with extra features:

  • Text styling options
  • Positioning cues on screen
  • Support for chapter markers

When you convert srt to vtt, you keep your text and timing but unlock new styling and accessibility benefits.

How to set up your Docker container

Turning this process into a Docker task means you can run it on any machine, with the same environment every time. Here is a plain-English setup:

  1. Install Docker on your system by following the guide at docker.com.
  2. Create a new folder for your project. Call it subtitle-converter.
  3. Inside the folder, add a file named Dockerfile.
  4. Choose an FFMPEG image. A good choice is jrottenberg/ffmpeg, which comes with audio and subtitle support built in.
  5. Write a short list of commands in the Dockerfile to install any extra tools you need.

Which base image should you choose?

You can pick a slim Alpine-based image or a full Ubuntu image. Alpine images tend to be smaller but may need extra setup. Ubuntu images are larger but often work out of the box.

  • For speed and size: jrottenberg/ffmpeg:4.3-alpine
  • For built-in support: jrottenberg/ffmpeg:4.3-ubuntu

What about FFMPEG in Docker?

FFMPEG does most of the work. Inside your container, you will use a command like:

ffmpeg -i input.srt output.vtt

This reads your SRT file and writes a VTT file. But you need to handle file paths and ensure your files are available inside the container. That’s where file mounting comes in.

How to convert SRT to VTT step by step

Let’s break down the process. You will:

  • Build your Docker image
  • Map your local folder into the container
  • Run FFMPEG to convert files
  • Check your output and tweak as needed

How does file mounting work?

When you run a Docker container, use the -v flag to link a folder on your computer to a folder inside the container.

  • Local folder: /Users/you/subtitle-converter
  • Container folder: /data

Example command:

docker run --rm -v /path/to/subtitle-converter:/data subtitle-image

Inside the container, your SRT lives in /data/input.srt and your VTT will appear at /data/output.vtt.

What are best practices and tips?

To make this process smooth every time, keep a few best practices in mind:

  • Name your files clearly: video-title_en.srt and video-title_en.vtt.
  • Check timecode formats: SRT uses commas, VTT uses periods (FFMPEG handles that for you).
  • Use the Subtitle editor in HappyScribe to preview your VTT with styling options.
  • Automate bulk tasks with a shell script or the HappyScribe API if you have many files.
  • Run a quick sanity check in your browser’s dev tools to verify your VTT tracks load correctly.

Pitfalls to avoid:

  • Missing file mappings. Double-check your -v paths.
  • Outdated FFMPEG builds. Pull the latest image for new features.
  • Broken timestamps after editing. Always open the VTT in a text editor for a quick review.
