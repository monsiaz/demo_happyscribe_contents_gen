Why am I getting timecode errors when I try to convert srt to vtt?

One of the top frustrations when you convert srt to vtt is timecode errors. You may see messages like “Invalid time format” or find that your captions don’t line up with the media. Let’s look at two common causes.

Are your timecodes missing milliseconds?

SRT files often use commas between seconds and milliseconds (for example, 00:00:05,500). In contrast, VTT requires a period (for example, 00:00:05.500). If you try a straight rename of .srt to .vtt, the file will fail.

# Using sed to fix commas in timecodes
sed -E 's/,/./g' input.srt > temp.vtt
echo "WEBVTT\n\n$(cat temp.vtt)" > output.vtt
  • Tip: Always add the “WEBVTT” header at the top, followed by a blank line.
  • Pitfall: Replacing all commas can affect text if you have commas in your dialogue. Use a more precise regex if needed.

Are there stray spaces or missing zeros?

Timecodes must follow HH:MM:SS.MMM exactly. If your SRT file has “0:0:5.5” or “00:00:05.50”, conversion tools will complain. You can standardize your timecodes with a script:

# Python script using webvtt-py for cleanup
from webvtt import WebVTT

vtt = WebVTT()
for caption in WebVTT().read('input.srt'):
    vtt.captions.append(caption)
vtt.save('output.vtt')

Using webvtt-py handles most formatting fixes for you. Learn more in our Subtitle editor guide.

How can I remove invalid characters in my SRT before conversion?

Sometimes you’ll see odd symbols or extra tags when you convert srt to vtt. These can come from non-UTF8 text, HTML tags, or hidden control characters. Here’s how to clean them up.

What tools help strip unwanted tags?

  • FFMPEG: Use its subtitle filters to remove styling.
  • iconv: Re-encode your file to UTF-8.
  • Regex: Find and remove HTML tags.
# Convert to UTF-8 and remove HTML tags
iconv -f ISO-8859-1 -t UTF-8 input.srt | sed -E 's/<[^>]+>//g' > clean.srt
  • Tip: Keep a backup of the original file so you can recover any lost context.
  • Pitfall: Removing tags can strip styling you might want. Consider exporting a human-made subtitles version if style matters.

What are the best tools for a smooth srt to vtt conversion?

Not all conversion methods are equal. Below are three ways to convert srt to vtt with minimal fuss.

FFMPEG command line

# Basic FFMPEG conversion

  • Pros: Fast, no extra installs if you already have FFMPEG.
  • Cons: May skip the “WEBVTT” header; you have to add it manually.

webvtt-py library

How do I handle overlapping or missing captions?

When you convert srt to vtt, you might see captions that overlap or gaps where captions are missing. These issues can turn a simple conversion into a manual headache.

Why do overlaps happen?

Overlaps occur when two captions share the same start or end times. VTT does not allow exact overlaps. You’ll get errors like “Cue timings overlap.”

# Example overlap in SRT
2
00:00:05,000 --> 00:00:07,000
Hello world!

3
00:00:07,000 --> 00:00:09,000
Welcome back!

Both cues touch on 00:00:07. You can fix this manually or use a script:

from webvtt import WebVTT

vtt = WebVTT().read('input.srt')
# Shift the start of caption 3 by 1ms
vtt.captions[2].start = '00:00:07.001'
vtt.save('fixed.vtt')
  • Tip: Check for zero-length gaps too; sometimes a tiny overlap is easier to fix than a gap.

How can I fill in missing captions?

Gaps in subtitles can confuse viewers. You have two main approaches:

  1. Manually add a “silent” caption with the gap’s timecode.
  2. Use a subtitle editor like Subtitle editor to spot gaps visually.
00:00:07.000 --> 00:00:08.500
[silence]
