How to automate subtitle workflows with API

In this article, you’ll learn how to convert SRT to VTT automatically and build a smooth subtitle workflow. We’ll walk through FFMPEG commands, a Python script with webvtt-py, and how to use the HappyScribe API for automatic transcription and human-made subtitles. By the end, you’ll know how to set up a pipeline that saves time and keeps your videos accessible.

What are SRT and VTT subtitle formats?

SRT and VTT are two popular subtitle formats:

  • SRT (SubRip Text): Simple text files with sequential cues and timecodes.
  • VTT (WebVTT): Similar to SRT but designed for HTML5 video players. Supports styling, positioning, and metadata.

Why convert SRT to VTT? Many online players and livestream platforms require VTT. If you have legacy SRT files, conversion is a quick step to stay compliant and accessible.

How can I convert srt to vtt with FFMPEG?

What command do I need to use?

FFMPEG is a free, open source toolkit for video and audio processing. You can also use it to convert subtitle formats. Here’s a simple command:

ffmpeg -i subtitles.srt subtitles.vtt

This reads subtitles.srt and outputs subtitles.vtt. If you need to embed the VTT in a video file, try:

ffmpeg -i input.mp4 -i subtitles.vtt -c copy -c:s webvtt output.mp4

Best-practice tips for FFMPEG conversion

  • Always check the timecodes after conversion. A quick spot check in VLC or your favorite player helps catch sync issues.
  • If your SRT has non-UTF-8 characters, add -sub_charenc UTF-8 to your command.
  • To batch convert multiple files, wrap the command in a shell loop:
for f in *.srt; do
  ffmpeg -i "$f" "${f%.srt}.vtt"
done

How do I use webvtt-py to convert srt to vtt?

Python script example

webvtt-py is a light Python library that reads and writes WebVTT. It’s great if you need more control in a script or app.

from webvtt import WebVTT

# Read the SRT file
vtt = WebVTT().read('subtitles.srt')

# Write out a WebVTT file
vtt.save('subtitles.vtt')

This code loads subtitles.srt and produces subtitles.vtt. It handles basic cue formatting for you.

Pitfalls to avoid with webvtt-py

  • Make sure your SRT timecodes are valid. webvtt-py won’t correct overlaps or missing fields.
  • If you have HTML tags or special characters, test how your video player handles them. Some browsers strip styling tags.

How can I build an automation with the HappyScribe API?

HappyScribe offers an API that makes it easy to get high-quality subtitles without manual steps. You can request automatic transcription, get a VTT file, or even order human-made subtitles.

How to send a request for automatic transcription

First, install a command-line HTTP client like cURL or use a library like requests in Python. Here’s a cURL example:

curl -X POST https://api.happyscribe.com/v1/transcriptions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "audio_url=https://example.com/video.mp4" \
  -F "subtitles_format=vtt"

This tells HappyScribe to transcribe the media at audio_url and return a VTT file.

How to download your VTT file via API

Once the job is complete, you’ll get a job ID in the response. Use it to fetch the result:

curl -X GET https://api.happyscribe.com/v1/transcriptions/JOB_ID \
  -H "Authorization: Bearer YOUR_API_KEY"

Look for the subtitles.vtt link in the JSON response and download it. You can then feed it to your CDN or video player.

How to request human-made subtitles

If you need higher accuracy, you can order human-made subtitles by changing one parameter:

curl -X POST https://api.happyscribe.com/v1/transcriptions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "audio_url=https://example.com/video.mp4" \
  -F "subtitles_format=vtt" \
  -F "model=human"

The API then notifies you when your VTT is ready, with 99% accuracy. You can manage and fine-tune your files in our Subtitle editor.

What are best practices and pitfalls to avoid?

To get the most out of your automated subtitle workflow, keep these tips in mind:

  • Standardize your file names: Use consistent naming like video01.mp4, video01.srt, video01.vtt. It helps in batch scripts and cloud storage.
  • Validate formats early: Check timecodes and encoding before running a large batch. One broken file should not stop the whole process.
  • Choose the right transcription model: Use automatic transcription for speed and human-made subtitles for high accuracy. You can even split by video length or topic.
  • Monitor API usage: Track your HappyScribe API calls to avoid unexpected costs. Set up alerts or usage caps.
  • Use the Subtitle editor: After you get your VTT, you can polish timing and text in our web-based editor. This step makes your subtitles viewer-friendly.
