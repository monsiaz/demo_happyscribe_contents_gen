What are the key differences between SRT and VTT?

Knowing the basic structure of each format helps when you do a manual conversion.

  • SRT format: Each cue is numbered, uses commas for milliseconds (00:00:01,500), and has no header.
  • VTT format: Starts with a “WEBVTT” header, uses periods for milliseconds (00:00:01.500), and supports styling tags like <v> for speakers.

Example of an SRT cue:

1
00:00:05,000 --> 00:00:07,000
Hello, world!

Example of the same cue in VTT:

WEBVTT

00:00:05.000 --> 00:00:07.000
 Hello, world!

Tip: If you leave the numbering in the VTT file, most players ignore it. But extra lines can slow down loading in some browsers.

How can you convert SRT to VTT using built-in Mac tools?

You can handle the conversion in two simple ways: a straight edit in TextEdit or a one-click Automator workflow.

Method 1: Quick edit with TextEdit

  1. Right-click your .srt file and open with TextEdit. If it’s rich text, go to Format > Make Plain Text.
  2. At the very top of the file, type WEBVTT on its own line, then add a blank line.
  3. Change commas in time codes to periods (find “,” replace with “.” in times only).
  4. Save and rename the extension from .srt to .vtt.

Pitfall to avoid: If you forget the blank line after the header, some video players will show no captions at all.

Method 2: Automator quick action

  1. Open Automator and choose “Quick Action.”
  2. Set “Workflow receives current” to “files or folders” in “Finder.”
  3. Add a “Run Shell Script” action with:
    4. awk 'NR==1{print "WEBVTT\n"}1'
  4. Save the Quick Action as “Convert to VTT.”
  5. In Finder, right-click any .srt and pick Quick Actions > Convert to VTT.

Best-practice tip: Test the first file by dragging it into QuickTime or VLC to confirm timing and header are correct.

How to test your converted VTT in different video players?

Not all players parse VTT the same way. Here’s a quick checklist:

  • QuickTime: Drag the .vtt into a video with the same name (video.mp4 & video.vtt).
  • VLC: Open your video, go to Subtitle > Add Subtitle File, and pick the .vtt.
  • YouTube: Upload video, then in Subtitles & CC, add a new track and upload the .vtt.
  • Browsers: Host the .vtt on a web server and reference it in <track kind="subtitles" src="xxx.vtt" srclang="en">.

Pitfall: Time code format errors often break loading. Double-check your separators are correct.

What should you watch out for when editing your VTT file?

Even after conversion, small mistakes can block captions from showing up. Here are key points:

  • Header: Always “WEBVTT” on line 1, then a blank line.
  • Timing: Use periods for milliseconds (no commas).
  • Empty cues: Remove lines with no text or no time ranges.
  • Styling tags: Open and close tags like <i>, <c>, <v> correctly.

Example clean cue:

00:01:15.200 --> 00:01:17.500

How to keep track of versions and naming conventions?

When you convert many files, version control helps avoid confusion. Here’s a simple rule set:

  • Use lower-case and no spaces: video-name.en.vtt.
  • Add a version number if you revise: video-name.v2.en.vtt.
  • Keep SRT and VTT in separate folders or label them clearly.
  • Log changes in a shared doc: date, editor, notes (e.g. “fixed timing for segment 4”).

Best-practice tip: Integrate file naming into your project template. That way every team member follows the same pattern.
