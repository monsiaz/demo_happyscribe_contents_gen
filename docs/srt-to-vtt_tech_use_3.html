Multilingual subtitle pipeline for global releases

In a world where content travels instantly across borders, video teams need a reliable pipeline to convert subtitle files from one format to another. A typical scenario involves a global marketing department that delivers an English SRT file for a new product launch. That file needs to be converted to a timed WebVTT file, enriched with language codes, and pushed to a content delivery network for viewers around the world.

This workflow matters because inconsistent subtitle formats can break automated player integrations, delay release schedules, and frustrate localization partners. By building a programmatic conversion pipeline, engineering teams free localization managers from manual edits and ensure every region gets high-quality captions on time.

Implementation steps

  1. Prepare your environment with an API key. Store it securely as an environment variable: 
    export HAPPYSCRIBE_API_KEY="your_api_key_here"
  2. Upload your source SRT file. If the file is public, skip to step 4. Otherwise request a presigned URL and upload via PUT: 
    # Get upload URL
curl -H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
"https://www.happyscribe.com/api/v1/uploads/new?filename=video_en.srt" \
| jq -r .signedUrl > upload_url.txt

# Upload file
curl -X PUT -T video_en.srt "$(cat upload_url.txt)"
  3. Confirm that the file is accessible by polling if needed. The presigned URL is valid for 15 minutes. If expired, repeat step 2.
  4. Create a transcription record in subtitle mode. Use your presigned URL or public URL: 
    curl -X POST \
-H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
-H "Content-Type: application/json" \
-d '{
  "transcription": {
    "name": "Global launch English",
    "language": "en-GB",
    "tmp_url": "'"$(cat upload_url.txt)"'",
    "is_subtitle": true,
    "service": "auto"
  }
}' \
https://www.happyscribe.com/api/v1/transcriptions
  5. Poll the transcription status until itâ€™s ready for export: 
    TRANS_ID="abc123"
while true; do
  STATUS=$(curl -s \
    -H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
    https://www.happyscribe.com/api/v1/transcriptions/$TRANS_ID \
    | jq -r .state)
  echo "Status: $STATUS"
  [ "$STATUS" = "automatic_done" ] && break
  sleep 10
done
  6. Request an export to WebVTT. You can export multiple formats in parallel by sending separate requests: 
    curl -X POST \
-H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
-H "Content-Type: application/json" \
-d '{
  "export": {
    "format": "vtt",
    "transcription_ids": ["'"$TRANS_ID"'"]
  }
}' \
https://www.happyscribe.com/api/v1/exports
  7. Poll the export endpoint for readiness and download: 
    EXP_ID="exp456"
while true; do
  EXP=$(curl -s \
    -H "Authorization: Bearer $HAPPYSCRIBE_API_KEY" \
    https://www.happyscribe.com/api/v1/exports/$EXP_ID)
  if [ "$(echo "$EXP" | jq -r .state)" = "ready" ]; then
    curl -L "$(echo "$EXP" | jq -r .download_link)" -o subtitles.vtt
    break
  fi
  sleep 5
done
  8. Validate the final WebVTT file for syntax and timing. You can run a quick check with standard WebVTT linters or upload to a staging player.
  9. Automate this sequence in your CI/CD pipeline or backend job scheduler. Use exponential back-off and error handling to retry on HTTP 429 or 5xx responses.

Mini FAQ

Which subtitle formats are supported?

HappyScribe supports SRT, VTT, STL, AVID, and many broadcast formats including Subtitle Workshop and Interop STL. You can request multiple formats in parallel.

How do I handle rate limits in bulk processing?

The default rate limit is 200 POST requests to /transcriptions per hour per API key. On HTTP 429, parse the retry_in_seconds field and back off accordingly. You can request limit increases for large-scale workflows.

Can I translate subtitles as part of the same pipeline?

Yes. After your initial export, use the /orders/translation endpoint to machine- or human-translate into multiple target languages. You can then export each translation to VTT or other formats in one step.
