How do you convert SRT to VTT with HappyScribe?

We’ll walk through a clear, step-by-step workflow. You’ll see how to set up your environment, upload or pull existing SRT files, call the conversion endpoint, and get your VTT files ready for deployment.

Step 1: gather your tools and credentials

  • HappyScribe account with API access (get your API key from your dashboard)
  • Your original .srt files (exported from your translation team or generated with our AI notetaker)
  • Command line tool (cURL) or a JavaScript runtime (Node.js) installed
  • Basic text editor for reviewing the output

Step 2: upload or reference your SRT files

You have two options:

  • Upload via API: Send each .srt file to HappyScribe. You’ll get a file ID back that you can use in conversions.
  • Reference an existing file: If you already processed the file for transcription or translation on HappyScribe, you can skip the upload and use the saved file ID.

Here’s how an upload call looks with cURL:


curl -X POST https://api.happyscribe.com/v1/files \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@/path/to/your/video.srt" \
  -F "type=subtitle"

Response example:


{
  "file_id": "12345abcde",
  "status": "uploaded"
}

Step 3: call the convert endpoint

Now that you have a file_id, request conversion to VTT. HappyScribe handles timecodes, cues, styling, and speaker tags automatically.


curl -X POST https://api.happyscribe.com/v1/files/12345abcde/convert \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "source_format": "srt",
    "target_format": "vtt"
  }'

Or with Node.js:


import fetch from 'node-fetch';

async function convertSrtToVtt(fileId) {
  const response = await fetch(
    `https://api.happyscribe.com/v1/files/${fileId}/convert`,
    {
      method: 'POST',
      headers: {
        Authorization: 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        source_format: 'srt',
        target_format: 'vtt'
      })
    }
  );
  const data = await response.json();
  console.log('Conversion started:', data);
}

convertSrtToVtt('12345abcde');

The API will return a job status and a link to download the VTT file once it’s ready.

Step 4: download and review your VTT file

Use the download link from the conversion response:


curl -X GET "https://api.happyscribe.com/v1/files/12345abcde/download?format=vtt" \
  -H "Authorization: Bearer YOUR_API_KEY" -o captions.vtt

Open captions.vtt in your text editor. You’ll see cues in this format:


WEBVTT

00:00:05.000 --> 00:00:10.000 align:start position:0%
Speaker 1: Welcome to our product demo.

00:00:12.000 --> 00:00:16.500 align:end position:100%
Speaker 2: Let’s dive into the main features.

Step 5: fine-tune in the subtitle editor

Maybe your timecodes need a nudge, or you want to style the captions. Load your VTT into HappyScribe’s subtitle editor:

  • Adjust start and end times by dragging cues on the waveform view
  • Add custom CSS classes or speaker labels
  • Split or merge captions for clarity
