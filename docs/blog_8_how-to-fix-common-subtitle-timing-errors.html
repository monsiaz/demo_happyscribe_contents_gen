How to fix common subtitle timing errors

In this article, you’ll learn how to identify and fix the most common subtitle timing errors, and how to convert srt to vtt formats without breaking your sync. We’ll dive into real command examples using FFMPEG, webvtt-py, and the HappyScribe API. You’ll also pick up best-practice tips and see how our Subtitle editor and Automatic transcription tools can save you time.

Why subtitle timing errors happen

Subtitle timing errors can spoil the viewer experience. They usually fall into two categories:

  • Offset errors: All lines appear too early or too late.
  • Drift errors: Sync shifts gradually over the video.

Common causes include:

  • Frame rate mismatch (23.976 vs 25 fps)
  • Manual edits without re-syncing
  • Incorrect time code conversion
  • Using the wrong subtitle format for your player

Tip: Always check your video’s frame rate before you start editing subtitles.

How do you shift subtitles using FFMPEG?

If all your subtitles are off by a fixed amount, you can apply a global shift. Here’s how:

ffmpeg -itsoffset 0.5 -i input.srt -c copy shifted.srt
  • This example moves subtitles 0.5 seconds forward. Use a negative value to move them back.
  • Replace input.srt with your file name.
  • Use -itsoffset before the input to apply the offset.

Pitfall: If you run -itsoffset after -i, nothing will change. The order matters.

What if your timing drifts over time?

Drift happens when the subtitle file’s time codes and video frame rate don’t match. You can fix this by stretching or compressing the subtitle timeline:

ffmpeg -i input.srt -filter_complex "[0:v]subtitles=input.srt:si=0:force_style='StartOpacity=0',setpts=PTS*1.002" -f null -
  • The setpts filter multiplies presentation timestamps. Adjust the factor (1.002) to match your drift.
  • Test small changes until the end of the video syncs correctly.

Tip: Export a section of the video and subtitles to test drift corrections before you process the full file.

How to convert srt to vtt after fixing timing

Once your SRT file is synced, you often need to convert srt to vtt for HTML5 players. Here are three quick methods:

Using FFMPEG

ffmpeg -i synced.srt synced.vtt
  • This single command reads the SRT and writes a valid WebVTT file.

Using webvtt-py

pip install webvtt-py
python -m webvtt srt2vtt synced.srt synced.vtt
  • Great for Python users. This script cleans up line breaks and adds the WEBVTT header.

Best practices and pitfalls to avoid

  • Always back up your original SRT before you start editing.
  • Check for negative time codes. Remove any lines that start before 00:00:00.000.
  • Watch out for overlapping subtitles. No two subtitles should share the same time window.
  • Use UTF-8 encoding to avoid character issues in WebVTT.
  • Preview your final VTT in multiple browsers to catch rendering quirks.

Tip: Use a media player like VLC or an online tool to preview your subtitles before distribution.
